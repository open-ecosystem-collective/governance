## How decisions are made

We use **Lazy Consensus** – the same model used by Apache Software Foundation.

> A proposal is accepted if no one objects within **72 hours**.
> Silence means agreement. Objections require a reason and an alternative.

Amendments to core values require a supermajority (2/3 of active members).
Tania holds a founder veto on core value changes for the first 3 years.

→ Full detail in [`decision-making.md`](decision-making.md)

## Revenue split

Every mandate delivered through the collective follows this split:

```
70%  →  Network member (the person who did the work)
20%  →  Vonarburg-Romero Consulting (brand, mandate sourcing, quality assurance, tooling, invoicing)
10%  →  Open Ecosystem (Lua Chea) (methodology, community)

```

This is non-negotiable and applies to all engagements equally.
It is documented here so every member and every client can see it.

## Want to contribute to governance?

Governance documents evolve. If you see something to improve:

1. Open an **Issue** describing what and why
2. Or submit a **Pull Request** with the proposed change
3. 72h Lazy Consensus applies
4. Merged changes are tagged as a new version in `CHANGELOG.md`

We welcome contributions from collective members and the broader community.

## Versioning

All governance documents follow **Semantic Versioning**:

- `v1.x` – clarifications and improvements, no structural change
- `v2.0` – new core values or significant structural change
- Every version is permanently tagged and available in the repository history
